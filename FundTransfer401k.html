401(k) Fund Transfer

401(k) money poseted in LUMS, both EE and ER, is held in a pending bucket to be funded to individual Prudential Retirement accounts every Wednesday. Sometimes Tom will request that an out of cycle transfer be done to close the calendar month. A major warning, the LUMS produced suporting docs cannot be recreated once you are done. If you do not save them they are gone forever. Getting John Graas to undo the file build will likely take 2 days. The timing is funding request on Wedenesday @ 10:00am means ACH Debit on the Friday.

Processing a 401(k) Funds Transfer with Squee

Here are the steps for sending a funding request (transfer file build) to Prudential Retirement. You must have authorization to send secure mail to Payroll Express as well as access the Prudential Sponsor Center website. Also, have your LUMS Helper open as it will make certain steps much easier.

Instructions are color coded:
Instructions or steps to be done in LUMS look like this.
Instructions or steps to be done in Squee look like this.

  1. Create a new Transfer in Squee

    • Click the Prudential button in the form navigator.
    • Click on the pink Add New Transfer button.
      1. A pop-up form with an anticipated transfer date will appear and ask you to confirm. At that time you can can change the date if you need to.
      2. Make sure the correct transfer is selected in the transfers list box.

  2. In Squee add ER batches that are ready to transfer.

    • Click on the ER Log tab.
    • Double click on a batch in the Pending list box to move it to the Assigned list box. Double click any Assigned batch to send it back to Pending.
      1. Squee will let you know with a pop-up message if all of the reports in the batch are being added to this transfer. (should i take this part out? and remove the pop-up notification in Squee?)
        2.  In Squee you are assigning "batches", but in the background Squee is only including reports that have not been in a previous Prudential funds transfer and has a LUMS Report Number signifying the report was uploaded to LUMS.

  3. In Squee add EE batches that are ready to transfer

    • Click on the EE tab.
    • Follow the same steps as adding ER batches.

  4. Check that RA Edit workspace is empty

    • LUMS Navigation: Main Menu / Labor Union Processing / Receipts & Adjustments / Receipts & Adjustments Edit
    • Keystrokes to type: 4 => {Enter}
    • If the report runs and a .pdf is created then there is an EE batch that was not updated and needs to be taken care of first.
      •  Really, nothing happening for this step is good. That's just how LUMS communicates sometimes.

  5. In LUMS run the Fund Detail Listing

    • LUMS Navigation: Main Menu / Labor Union Processing / Fund Detail Processing / Fund Detail Listing
    • Keystrokes to type: 4 => {Enter} => S {Enter} => Y => {Enter} V => {Enter}
    • A .pdf of the report will open. DO NOT CLOSE it, we need to save it later.
    • The LUMS report total must match the Squee transfer total.
    • This is the 1st run of this report, we will be doing one more after the file build.
      •  The total is on the second to the last and last page.

  6. Check that Squee and LUMS are in agreement for the transfer

    • In LUMS run the Fund Detail Listing for the 401(k) fund pending money.
    • Check the LUMS total against the Squee total.
    • If Squee and LUMS are in agreement continue with process. If not follow the Pru Transfer Troubleshooting Guide

  7. In LUMS run the transfer file build (401k Enrollment - Prudential)

    • LUMS Navigation: Main Menu / CIGNA Transfers / 401k Enrollment - Prudential
    • Keystrokes to type: [today's Month(2 digits)][today's Year(2 or 4 digits)] => {Enter} => [the number that corresponds to Last EE PPE date in Squee] => {Enter} => Y => {Enter} => V => {Enter} => Y => {Enter}
    • note to self: need to explain more on the last few keystrokes. need to explain how to abort (remove from RA Edit) a bad transfer file build.
    • A .pdf of the report will open. DO NOT CLOSE it, we need to save it later.
    • There are three totals on this report that need to go into Squee on the Details tab. These fields in have labels with a green background and white letters and the input boxes have a thin black border around them.
      1. Put Employer 401k for [work month] amount into the LUMS ER field
      2. Put Participant 401k for [work month] amount into the LUMS ER field
      3. Put Employer 401k Hours for [work month] amount into the LUMS R/A Hours field
    • The Discrepancy fields should be $0.00. If not you have a problem.

  8. In LUMS run the build verification (Prudential 401(k) Verification

    • LUMS Navigation: Main Menu / CIGNA Transfers / Prudential 401(k) Verification
    • Keystrokes to type: [today's Month(2 digits)][today's Year(2 or 4 digits)]
    • With LUMS Helper open, hit Ctrl & P.
      1. An MS Word document will open with a PrintScreen of the LUMS window results for backup.
    • With the same LUMS form in focus, hit {Enter}.
      1. The LUMS window should close. That is the good scenario.
        2.  Another case of LUMS communicating 'nothing' actually means 'perfect'.
      2. If a .pdf pops-up, that is bad. That means there is a net debit for the member or members on the .pdf report. Net debits or take backs can only be sent as a special request diretly to our Prudential service representative and cannot be included in a file build.

  9. In LUMS run Receipts and Adjustments Edit

    • LUMS Navigation: Main Menu / Labor Union Processing / Receipts & Adjustments / Receipts & Adjustments Edit
    • Keystrokes to type: {Enter} => {Enter} => {Enter}
    • A .pdf of the report will open. DO NOT CLOSE it, we need to save it later.
    • You can check the total if you want but it shouldn't matter

  10. In LUMS run Receipts and Adjustments Update

    • This step will make the file build permanent. If you closed these supporting docs from LUMS they cannot be recreated after this step: 1st Fund Detail Listing, File Build, Build Verification and RA Edit.

    • LUMS Navigation: Main Menu / Labor Union Processing / Receipts & Adjustments / Receipts & Adjustments Update
    • Keystrokes to type: {Enter} => {Enter} => {Enter} => [the R/A Confirm total from Squee]
      •   If you have LUMS Helper open you can just copy and paste the verification amount must be a negative number.
    • A .pdf of the report will open. DO NOT CLOSE it, we need to save it later.

  11. Run the Fund Detail Listing again

    1. LUMS Navigation: Main Menu / Labor Union Processing / Fund Detail Processing / Fund Detail Listing
    2. Keystrokes to type: 4 => {Enter} => S {Enter} => Y => {Enter} V => {Enter}
    3. A .pdf of the report will open. DO NOT CLOSE it, we need to save it later.

  12. Save All the Supporting Docs

    13. All suporting docs are stored in the network folder:
    \\HBAI2012SRVR\transfer\401(k)\Prudential File Transmissions\Prudential 401k Weekly Transfers\[Current Year] Prudential Transfers

    • Now that you are done with LUMS, the backup documents produced now need to be saved.to save all the supporting documents. It is important to have LUMS Helper open and use the hotstrings to fill in the file save names. Hotstrings are like autoreplace in MS Word, only better. One of the hotstrings triggers a backup save of the file build. Since these are irreplaceable, this is not being overly cautious. Save the files in this order:
    Backup file Hotstring Printing
    1. 1st Fund Detail listing pdf fdls1 All Pages
    2. Transfer File Build pdf fb1 All Pages
    3. Receipts & Adjustments Edit pdf rae1 The Last Page
    4. Receipts & Adjustments Update pdf rau1 All Pages
    5. 2nd Fund Detail listing pdf fdle1 All Pages
    6. Build verification doc bv1 All Pages

  13. This would be when you do lost earnings

      14. skipping this for now.

  14. Send Tom an Email and Give Him a Copy

    • Open a blank email.
    • In the 'To' line type in "tompayexp"
      1. LUMS helper will create the template email body.
    • In Squee at the bottom right of the Prudential form you will see a long wide purple button titled Copy Totals to Clipboard. Click it to copy the totals in the box right below it.
    • Go back to the email and paste where it says [Paste Squee totals here].
    • Send the email to Tom.
    • Print a copy of the sent email and hand to him as well.

  15. Send email to let everyone know it is ok to key batches into LUMS now

    • Open a new email in Outlook.
    • In the To: line type 'trasarpau'.
      1. LUMS Helper will autofill the email.

  16. At 10:00 a.m. Send Funding Request by Secure Mail

    17. Sending the transfer file earlier could very well result in the funding request settling the next day, which does not give Tom enough time to do the funds transfer with First Hawaiian Bank. Much later and the ACH debit may settle in 3 business days. On the normal cycle that would be a Monday. Issues with closing the month?

    • Go to secure mail.
    • In the 'To' line type in "payexp" and LUMS Helper will type out the email.
    • Click on the browse button icon to attach the transfer file.
      1. A pop-up file explorer will appear.
      2. Click on the 'Browse' button.
      3. Navigate to the same folder that the backups were stored.
      4. Select the file called 767258CC that was created today.
      5. On the pop-up file explorer click the 'Add' button.
      6. Once the file is attached (it takes a while, about a minute) click the done button.
    • Click on the attachment to double check it is the right file
      1. Scroll to the bottom and double check the total is matching.
    • In the To field type "pex", yes another hotstring for payroll.express@prudential.com.
    • change to cc address to your email address.

  17. You will receive 2 email confirmations

    18. You get two email confirmations. One a few minutes after you send the secure mail. This only confirms that they received the email and attachment but it is not confirming the request is good and that the ACH Debit will be set up. That comes about 2 hours later if all has gone as planned.

    • Email confirmation from payroll express
      1. We just print and add to the hard copies.
    • email from Prudential funding?
      1. This is a secure email, you need to login to secure mail to download it. This one has the transfer request total and this one we save as backup. When you save as into the backup folder type in the hotsting "prufun".
      2. Back to Prutential, check the box at the top left of Details tab. It is the check box to the left of 'Sent To Prudential'.
        3.  This step turns off the alert in Squee asking if the transfer file was sent. It also triggers a function that changes the file named 767258CC to 767258CC_sent_[today's date].

  18. Send Email to Tom with ACH Debit Date

    19. When you get the funding email, you will also be able to see the funiding shit on the pru website.

    • In the 'To' line type "tomach"... and you know what happens next.
